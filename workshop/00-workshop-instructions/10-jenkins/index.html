Export Formats After an application has been packaged into a Habitat Artifact, the .HART file can be run directly by the Supervisor in your environment of choice. The immutable packages built in the Studio clean-room can be expected to run exactly as they were tested. This way application developers can focus on building for functionality and features, instead of spending time tuning an app to the chosen deployment environment. In CI/CD build and release pipelines, you may have a requirement that a package be tested in a different file format, such as a Docker container. The Habitat Studio supports exporting any built .HART artifact to a variety of formats, such as Docker, Mesos, CloudFoundry, ACI, or tar. This way Habitat can seamlessly integrate into your Build pipeline without disrupting the testing tools a team is already using. In this class we'll test the packages we build using Docker. Now that you have some background on the components of Habitat we'll be using in this course, let's put hands to keyboard and learn how to use some of the tools. We'll start by installing the Habitat command-line-interface on your workstation.
