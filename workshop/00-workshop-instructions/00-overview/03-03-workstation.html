Workshop Workstation

Directories:

  • init
    Initalizes the local Habitat Builder/Depot with packages from the Chef Habitat Origin
  • servers
    Performs remote administration on all the Store devices
  • shim
    Contains basic Habitat services packages for each module used to pre-load remote devices with available packages
  • workshop
    Contains the code for each interactive module

Directory ~/Habitat-workshop

Scripts:

  • build-cache.sh
    Builds every service in the shim and workshop Directories. This allows all the required packages to be downloaded reducing the time spent waiting for packages to load during each module.
  • configure-demo.sh
    Builds and deploys all the workshop modules to the nodes
  • configure-workshop.sh
    Builds packages and configures the workshop. This will also publish and promote the base "shim" packages
  • disable-remote.sh
    Removes enviornment variables allowing you to interact with the local habitat supervisor
  • enable-remote.sh
    Sets local enviornment variables allowing you to interact with remote habitat supervisors
  • env.sh
    Sets common enviornment variables used by all the modules
  • prep-git.sh
    Connects to and downloads the jenkins module from the local git repo, changes the readme, commits and pushes the code. This will then trigger the jenkins build. (Note: must be run outside of the habitat studio)
  • purge.sh
    This removes all of the habitat build artifacts from the local package Directories

Directory ~/Habitat-workshop/init

Scripts:

  • migrate-chef-origin.sh
    This downloads specific packages from the public Habitat Builder from the Chef origin and uploads them to the local Habitat Builder. These are required by the effortless modules.

      • workshop/proxy [stable]

      • workshop/deploy_rollback [qa]

Directory ~/Habitat-workshop/workshop

Scripts:

  • build-all.sh
  • load-all.sh
  • publish-all.sh
  • unload-all.sh

Directory ~/Habitat-workshop/workshop/*/

Each workshop directory contains the same scripts:

  • demote.sh
    Removes a channel from the latest build (if it has been published)
  • load.sh
    Loads your local packages to your local Habitat Supervisor
  • promote.sh
    Adds a channel to the latest build (if it has been published)
  • publish.sh
    Publishes the latest build to the remote builder
  • unload.sh
    Unloads your local packages from your local Habitat Supervisor
Please note some modules may have additional module specific scripts
